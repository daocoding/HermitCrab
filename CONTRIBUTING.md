# Contributing to Build Your Own AI Assistant 🦀

Thank you for your interest in contributing! This project thrives on community input.

## How to Contribute

### 📝 Improve Documentation
- Fix typos, clarify explanations, improve examples
- Translate content to other languages
- Add diagrams or visual aids

### 💡 Share Your Bridge
Built a bridge for a platform we don't cover? We'd love to include it!
1. Follow the [Bridge Protocol](chapters/05-bridge-protocol.md) pattern
2. Create a chapter or example in the appropriate directory
3. Submit a PR

### 🐛 Report Issues
- Use GitHub Issues for bugs or unclear documentation
- Include your OS, Node.js version, and relevant config

### 🌐 Translations
We're actively building bilingual (EN/中文) content. Additional languages welcome!

## Guidelines

1. **Keep it practical** — Every addition should solve a real problem
2. **Show, don't tell** — Working code > theoretical explanations
3. **Stay opinionated** — This project has a clear architecture philosophy. PRs should align with the "One Shell, Many Crabs" approach
4. **Test your code** — Make sure examples actually run

## PR Process

1. Fork the repo
2. Create a branch (`feat/discord-bridge` or `fix/memory-overflow`)
3. Make your changes
4. Submit a PR with a clear description of what and why

## Code of Conduct

Be kind. Be helpful. Be the developer you'd want to collaborate with.
